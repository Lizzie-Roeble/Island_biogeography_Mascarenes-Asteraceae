Preparation: Improving the target file

Expanding and fixing the reference target file has been shown to be an important process in the read mapping step with Hybpiper (https://bsapubs.onlinelibrary.wiley.com/doi/10.1002/aps3.11420). Chris Jackson has expanded the angio353 target file (mega353) and developed some useful tools to improve/expand target files, check reading frame, check for stop codons, and remove low-complexity loci that significantly slow down the read mapping assembly. 

Resources and references: 
* https://github.com/mossmatters/HybPiper/wiki/Troubleshooting,-common-issues,-and-recommendations#14-fixing-and-filtering-your-target-file
* https://github.com/mossmatters/HybPiper/wiki/Full-pipeline-parameters#70-hybpiper-check_targetfile
* https://github.com/mossmatters/HybPiper/wiki/Full-pipeline-parameters#80-hybpiper-fix_targetfile

Below is the code used to improve our target file. 
This was run from /Target_file_improvement on the interactive node

Preparation: Organizing and downloading sequence data

We are combining sequences from two data sources: 
1. Our new island+mainland samples that were sequenced with comp1061 and angio353
2. PAFTOL/GAP sequences that include Calyceraceae outgroups, species from all tribes in Asteraceae, and species around our target lineages 

In 0_raw_sequneces we have two subdirectories: 
   /0_raw_sequneces/Island
   /0_raw_sequneces/PAFTOL 

The list of PAFTOL sequnces is detailed in /namelists/PAFTOL_Asteraceae_sequences.xlsx. We start by downloading the PAFTOL sequneces from the ENA sequence downloader (https://github.com/enasequence/ena-ftp-downloader/) into our woring project directory on Habrok. 

The script does the following:
1. Sets file paths for this project (update for future runs as needed)
2. Runs the sequence downloader for the selected assessions (/namelists/PAFTOL_Asteraceae_accessions.tsv)
   * Note! These need to be saves as a tsv file- no header, single column 
3. Reorganize the sequences. Move R1 and R2 seqeunces out of the individual nested directories and into the PAFTOL directory
4. Rename the sequneces based on the "Name" column in PAFTOL_Asteraceae_sequences.csv. This calls a python script (rename_PAFTOL_sequences.py) to execute the rename. The name is standardized to be Family_Subfamily_Tribe_Genus_species_ENAid_Read#

Important note!! 
This step requires manual inspection. Some species (8/176 in this case) do not have a valid format=READS_FASTQ
So after the bulk download run ls -1 | grep '.gz' | wc -l to see how many samples didnt download. 
Then manually identify samples missing/not downloaded, go to the ENA or PAFTOL portal to manually download/upload to habrok, and manually rename. 
(I have trialed automating this step and its not worth the effort...easier to just manually deal with these species)

import csv
import os
import sys

# Define function to read CSV and rename files
def rename_sequences(output_location, seq_names):
    # Read CSV and rename files
    with open(seq_names, 'r') as csvfile:
        reader = csv.DictReader(csvfile)
        for row in reader:
            accession_id = row['Run Accession Number']
            name = row['Name']

            if accession_id and name:
                # Rename forward read
                old_name_1 = f"{accession_id}_1.fastq.gz"
                new_name_1 = f"{name}_1.fastq.gz"
                if os.path.exists(os.path.join(output_location, old_name_1)):
                    os.rename(os.path.join(output_location, old_name_1), os.path.join(output_location, new_name_1))
                    print(f"Renamed {old_name_1} to {new_name_1}")
                else:
                    print(f"Warning: {old_name_1} not found.")

                # Rename reverse read
                old_name_2 = f"{accession_id}_2.fastq.gz"
                new_name_2 = f"{name}_2.fastq.gz"
                if os.path.exists(os.path.join(output_location, old_name_2)):
                    os.rename(os.path.join(output_location, old_name_2), os.path.join(output_location, new_name_2))
                    print(f"Renamed {old_name_2} to {new_name_2}")
                else:
                    print(f"Warning: {old_name_2} not found.")

# Main function to handle command-line arguments
def main():
    if len(sys.argv) != 3:
        print("Usage: python3 rename_sequences.py <OUTPUT_LOCATION> <SEQ_NAMES>")
        sys.exit(1)

    output_location = sys.argv[1]
    seq_names = sys.argv[2]

    rename_sequences(output_location, seq_names)

if __name__ == "__main__":
    main()

    gunzip "$zip"

Step 2b: Get summary stats from Hybpiper 

For each of the sequence datasets (1.island, 2.PAFTOL, 3.island+PAFTOL) get summary stats pre-filtering.

Breakdown of the script:
* datadir=$PWD stores the location to the Hybpiper data (so run this script from the 2_Hybpiper directory)
* Change current working directory to local disk ($TMPDIR) in compute node for faster processing  
* For loop to extract Hybpiper tar files to $TMPDIR. 
   * First line in loop interates of all the .tar files in the "datadir" directory (ie 2_Hybpiper)
   * "tar xf $datadir/$samplename.tar" extracts the contents of the .tar file specified by $samplename to the current working directory ($TMPDIR)
* Then we get the summary stats. 
   * hybpiper stats command will then summarize target enrichment and gene recovery efficiency for a set of samples
      * We need to supply the reference file used for mapping the reads. The file is located in the target_files directory, which is one level up from the current datadir directory.
      * And specify the path to the namelist.txt file that contains the names of the samples. The file is located in the directory one level up from datadir.
   * hybpiper recovery_heatmap takes the sequence length table output of hybpiper stats and creates a figure to visualize the recovery efficiency. 
   * Then copy the summary results back to /scratch in a new directory called 2_Hybpiper_summary

Resources and references:
* https://github.com/mossmatters/HybPiper/wiki#hybpiper-stats

form /scratch/$USER/Hawaii_Mascarenes_Asteraceae/data/2_Hybpiper run:
sbatch /scratch/$USER/Hawaii_Mascarenes_Asteraceae/scripts/hybpiper_summary_stats.sh

#!/bin/bash

#SBATCH --time=2-00:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --mem=20GB
#SBATCH --job-name=hybpiper_summary_stats_all
ulimit -n 16384

# Run this script from the 2_Hybpiper folder

# Store location of data
datadir=$PWD

# load HybPiper2 module
module load MAFFT
module load Hybpiper/2.1.6-foss-2022b

# Change to local disk in compute node for faster processing
cd $TMPDIR

# Extract tar files to $TMPDIR
for samplename in $datadir/*.tar; do
  tar xf $samplename
done

# 1. Summary stats: Island

Step 3. Paralogy resolution (Paragone)

ParaGone is a Python package (with external dependencies) designed to identify ortholog groups from a set of paralog sequences from multiple taxa. This process has been decribed as both paralogy resolution and orthology inference. ParaGone implements multiple different algorithms to infer ortholog groups. These algorithms are applied to gene trees that contain paralogs, and ortholog groups are inferred based on algorithm-specific parsing of the tree topologies. The pipeline uses algorithms described and implemented by Yang and Smith 2014. These approaches have since been adapted for target capture datasets. (from github)

Resources and references:
* https://github.com/chrisjackson-pellicle/ParaGone
* https://github.com/chrisjackson-pellicle/ParaGone/wiki/Tutorial
* https://github.com/chrisjackson-pellicle/ParaGone/wiki
* https://academic.oup.com/mbe/article/31/11/3081/2925722

## Step 1. Align loci

# Run MAFFt with the auto settings

module load MAFFT/7.490-gompi-2021b-with-extensions

for file in *.FNA;
do
mafft --auto "$file" > ../../4_Alignments/Hybpiper/"${file%.FNA}"_aligned.fasta
done

# note the ${var%pattern} removes the .FNA ending giving just the loci base name and adding the _aligned.fasta ending

cd ../../4_Alignments/Hybpiper
find . -type f -size 0b -delete

## Step 2. Filter/clean 

# Remove alignment sites based on proportion of missing data (-p)

# Load phyx 
module load phyx/1.3-foss-2022a

for file in *_aligned.fasta;
do 
pxclsq -s $file -o "${file%_aligned.fasta}"_aligned_rm.fasta -p 0.70
done

# -p [proportion of data required to be present]
# here, 70% of the data required to be present by column, ie sites with >30% missing data removed

# Delete the unfiltered alignments 
rm *_aligned.fasta

# Remove alignments that are empty 
find . -type f -size 0b -delete

## Step 3. Summary statistics on alignment

# load python and activate virtual pyton environment with AMAS installed

source /home4/$USER/software/venvs/target_capture/bin/activate

python3 /home4/$USER/software/venvs/target_capture/lib/python3.9/site-packages/amas/AMAS.py summary --in-files  *_aligned_rm.fasta --in-format fasta --data-type dna

#!/bin/bash

#SBATCH --time=12:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --mem=15GB
#SBATCH --job-name=alignments_paralogs

# Run this from 4_Alignments/Paralogs/

# 1. Clean paralog sequence names and output into current directory (/4_Alignments/Paralogs/)
# We call Qin's renaming python script for cleaning up the Hybpiper appended info onto the sequence names
# This script splits the label at the space " " and keeps the paralog ids (.main, .0, etc) which we need to keep the paralog names unique for RAxML

module load Python/3.9.6-GCCcore-11.2.0
source /home4/$USER/software/venvs/target_capture/bin/activate

for file in ../../2_Hybpiper_paralogs/paralogs_all/*.fasta; 
do 
  filename=$(basename $file) 
  python3 /scratch/$USER/Hawaii_Mascarenes_Asteraceae/scripts/rename_fastafile.py $file $filename; 
done

# 2. Align loci
# Run MAFFt with the auto settings

module load MAFFT/7.490-gompi-2021b-with-extensions

for file in *.fasta;
do
  mafft --auto "$file" > "${file%.fasta}"_aligned.fasta
done
# note the ${var%pattern} removes the .fasta ending giving just the loci base name and adding the _aligned.fasta ending 

# Remove the original unaligned paralogs_all.fasta files, leaving paralogs_all_aligned.fasta
rm *paralogs_all.fasta
# Remove any alignments that are empty
find . -type f -size 0b -delete

# 3. Filter/clean alignments
# Remove alignment sites based on proportion of missing data (-p)

# Load phyx 
module load phyx/1.3-foss-2022a

for file in *_aligned.fasta;
do 
pxclsq -s $file -o "${file%_aligned.fasta}"_aligned_rm.fasta -p 0.70
done

# -p [proportion of data required to be present]
# here, 70% of the data required to be present by column, ie sites with >30% missing data removed

# Delete the unfiltered alignments 
rm *_aligned.fasta

# Remove alignments that are empty 
find . -type f -size 0b -delete

## 4. Summary statistics on alignment

python3 /home4/$USER/software/venvs/target_capture/lib/python3.9/site-packages/amas/AMAS.py summary --in-files  *_aligned_rm.fasta --in-format fasta --data-type dna

# deactivate virtual environment
deactivate

base_name=$(basename "$alignment" _aligned_rm.fasta)

# -m [substitutionModel][GTRGAMMA or GTRGAMMAI or GTRCAT]

# -N or-# [Specify the number of alternative runs on distinct starting trees][can specify auto with autoMRE]
# -x [randonm seed for bootstrap]
# -p [random seed for parsimony inferences]
# -T [number of threads]
# -k [specifiies that the bootstrap]
# -q [file with the partitions]

Tree Comparison: Robinson Foulds Distances

Robinson-Foulds (RF) distance is a metric used to compare the differences between two phylogenetic trees. It's a commonly used measure in computational biology and bioinformatics to assess the similarity or dissimilarity between trees. 

The RF distance measures the dissimilarity between two phylogenetic trees by counting the number of different splits (bipartitions) between the trees. A split in a tree divides the set of taxa (species or genes) into two non-overlapping groups. The RF distance essentially counts the number of splits that are present in one tree but not in the other.

The R script below computed the RF distances between the trees and outputs a heatmap plot to compare the trees, and we also perfomr PCA on the distance matrix to provide a visual representation of the overall similarity among the trees. 


Observations from the heatmap and PCA: 
   * Hybpiper (defualt), Paragone-MO, Paragone-RT form a cluster of similarity. This suggest Hybpiper default does a similar job to methods that actively resolve paralogs
   * Trees with the greatest dissimilarity: Paralogs and Paragone-PF. This makes sense...one actively incorporates information from paralogs wilte the other exclused loci with paralogs entirely
   * The paralog-free dataset (Paragone-PF) is distinct from paralogy resolution

#!/usr/bin/env Rscript

# Adjusted Robinson Foulds Distance

# Load necessary libraries
library(phangorn)
library(ape)
library(phytools)
library(ggplot2)
library(vegan)
library(reshape2)
library(viridis)
library(ggrepel)

# Read the trees
# Be sure path is correct
tr1 <- read.tree("../Hybpiper/Species_tree/Species_tree_hybpiper_wastral.tre")
tr2 <- read.tree("../Paragone/Species_tree/PF/Species_tree_paragonePF_wastral.tre")
tr3 <- read.tree("../Paragone/Species_tree/MO/Species_tree_paragoneMO_wastral.tre")
tr4 <- read.tree("../Paragone/Species_tree/RT/Species_tree_paragoneRT_wastral.tre")
tr5 <- read.tree("../Paralogs/Species_tree/Species_tree_paralogs_astralpro.tre")

# 1. Preparation 

## Root the trees
outgroup <- c("Calyceraceae_Calycera_crassifolia_ERR7618534", "Calyceraceae_Acicarpha_tribuloides_ERR4180184", "Calyceraceae_Boopis_anthemoides_ERR7618533")
outgroup4 <- c("Barnadesioideae_Barnadesieae_Schlechtendalia_luzulifolia_ERR7618367", "Barnadesioideae_Barnadesieae_Barnadesia_odorata_ERR5033727", "Barnadesioideae_Barnadesieae_Chuquiraga_straminea_ERR7618373")

tr1 <- root(tr1, outgroup = outgroup, resolve.root = TRUE)
tr2 <- root(tr2, outgroup = outgroup, resolve.root = TRUE)
tr3 <- root(tr3, outgroup = outgroup, resolve.root = TRUE)
tr4 <- root(tr4, outgroup = outgroup4, resolve.root = TRUE)
tr5 <- root(tr5, outgroup = outgroup, resolve.root = TRUE)

# Check if the trees are rooted
cat("Is tr1 rooted?", is.rooted(tr1), "\n")
cat("Is tr2 rooted?", is.rooted(tr2), "\n")
cat("Is tr3 rooted?", is.rooted(tr3), "\n")
cat("Is tr4 rooted?", is.rooted(tr4), "\n")
cat("Is tr5 rooted?", is.rooted(tr5), "\n")

# Remove the outgroup from trees 1:3 to match RT
species <- outgroup
pruned.tr1 <- drop.tip(tr1, species)
pruned.tr2 <- drop.tip(tr2, species)
pruned.tr3 <- drop.tip(tr3, species)
pruned.tr5 <- drop.tip(tr5, species)

# Check if the pruned trees are still rooted
cat("Is pruned.tr1 rooted?", is.rooted(pruned.tr1), "\n")
cat("Is pruned.tr2 rooted?", is.rooted(pruned.tr2), "\n")
cat("Is pruned.tr3 rooted?", is.rooted(pruned.tr3), "\n")
cat("Is pruned.tr5 rooted?", is.rooted(pruned.tr5), "\n")

# 2. Make a table of RFadj values

# Create an empty matrix with 4 rows and 4 columns
matrixinp = matrix(data=NA, nrow=5, ncol=5) 

# Display the empty matrix 
print(matrixinp)

# Make list of trees
tree_list <- list(pruned.tr1, pruned.tr2, pruned.tr3, tr4, pruned.tr5)

for(j in 1:length(tree_list)){ 
  for (i in 1:length(tree_list)) {
    tree1 <- tree_list[[i]]  
    tree2 <- tree_list[[j]]
    matrixinp[i,j] = RF.dist(tree1, tree2, check.labels = TRUE, normalize = TRUE, rooted = TRUE)
  }
} 

# Give the matrix column and row names
colnames(matrixinp) <- c("HybPiper", "ParaGone-PF", "ParaGone-MO", "ParaGone-RT", "Paralogs")
rownames(matrixinp) <- c("HybPiper", "ParaGone-PF", "ParaGone-MO", "ParaGone-RT", "Paralogs")

# View the matrix
matrixinp

# export the matrix as a csv
write.csv(matrixinp, "RFDistance_Hawaii_Mascarenes_Asteraceae_coalescent.csv")

# Plot the heatmap

# Melt the matrix to long format
matrix_long <- melt(matrixinp)

# Filter out the lower triangle and the diagonal by setting lower triangle values to NA
matrix_long <- matrix_long[as.numeric(matrix_long$Var1) <= as.numeric(matrix_long$Var2), ]

# Plot the heatmap with viridis color scale
heatmap_plot <- ggplot(data = matrix_long, aes(x = Var1, y = Var2, fill = value)) +
  geom_tile() +
  scale_fill_viridis(name = "RF Distance", option = "D", direction = 1) +
  theme_minimal() + 
  theme(axis.text.x = element_text(angle = 45, vjust = 1, 
                                   size = 10, hjust = 1),
        axis.text.y = element_text(size = 10),
        title = element_text(size = 10)) +
  labs(x = 'Tree Comparison', y = 'Tree Comparison', title = 'Heatmap of Adjusted Robinson-Foulds Distances')

# Print the heatmap
print(heatmap_plot)

ggsave(heatmap_plot,
       filename = "heatmap_RFDistance_Hawaii_Mascarenes_Asteraceae_coalescent.pdf", 
       device = "pdf", width = 5, height = 4, units = "in")


# 3. Visualize Robinson-Foulds values as a PCA
# Classical multidimensional scaling (MDS) of a data matrix. Also known as principal coordinates analysis:

pca <- pcoa(matrixinp, correction = "none", rn = NULL)

# Convert PCoA results to a data frame for plotting
pca_df <- as.data.frame(pca$vectors)
pca_df$Sample <- rownames(pca_df)  # Add sample names for labeling

# Plot PCA
pca_plot <- ggplot(pca_df, aes(x = Axis.1, y = Axis.2)) +
  geom_point(size = 3, aes(color = Sample)) +
  geom_text_repel(aes(label = Sample), size = 4, box.padding = 0.5, point.padding = 0.5, max.overlaps = Inf) +
  scale_color_manual(values = RColorBrewer::brewer.pal(n = length(unique(pca_df$Sample)), name = "Set1")) +  # Use a default ggplot2 color palette
  theme_bw() +  # Use theme_bw for a clean background
  theme(plot.margin = margin(10, 10, 10, 10),  # Ensure there's space around the plot
        axis.title = element_text(size = 14),
        axis.text = element_text(size = 12),
        legend.position = "none") +  # Remove legend
  labs(x = paste("PCoA Axis 1 (", round(pca$values$Relative_eigenvalues[1] * 100, 1), "%)", sep = ""),
       y = paste("PCoA Axis 2 (", round(pca$values$Relative_eigenvalues[2] * 100, 1), "%)", sep = ""),
       title = "PCA of Robinson-Foulds Distances")
pca_plot

# Save the PCA plot
ggsave(pca_plot, filename = "pca_RFDistance_Hawaii_Mascarenes_Asteraceae_coalescent.pdf", device = "pdf", width = 6, height = 5, units = "in")

# load phyx

cd Rooted_gene_trees/

# Be sure to update to correct path to match dataset! 
# Update the outgroup as needed - Here we root between Asteraceae and Calyceraceae
for file in ../../../5_Phylogeny_coalescent/Hybpiper/Gene_trees/RAxML_bipartitions.*_hybpiper;
do
    filename=$(basename "$file")  # Extracting just the file name without the path
    pxrr -t "$file" -g Calyceraceae_Calycera_crassifolia_ERR7618534,Calyceraceae_Boopis_anthemoides_ERR7618533,Calyceraceae_Acicarpha_tribuloides_ERR4180184 -o "rooted_${filename}" -s
done

# Step 2. Sorta date

cd ../SortaDate/
module load Python/3.9.6-GCCcore-11.2.0

#2a. Get the tip-to-root variation with the get_var_length.py
# get_var_length.py DIR [-h] [--flend FILEENDING] [--outg OUTGROUPS] [--loc PHYX_LOCATION] [--outf OUTF]

python /home4/$USER/software/SortaDate/src/get_var_length.py ../Rooted_gene_trees/ --flend _hybpiper --outg Calyceraceae_Calycera_crassifolia_ERR7618534,Calyceraceae_Boopis_anthemoides_ERR7618533,Calyceraceae_Acicarpha_tribuloides_ERR4180184 --outf var_length

#2b. Get the bipartition support with get_bp_genetrees.py
# get_bp_genetrees.py [-h] [--flend FILEENDING] [--loc PHYX_LOCATION] [--outf OUTF] DIR SPECIESTREE
# note here: species tree is wASTRAL with min at 0.33

# We will filter out these loci in order for combine_results.py to run smoothly
awk '/NA/ {print $1}' var_length > skipped_genes.txt
awk 'NR==FNR {skipped[$1]; next} !($1 in skipped)' skipped_genes.txt var_length > var_length_filtered
awk 'NR==FNR {skipped[$1]; next} !($1 in skipped)' skipped_genes.txt bp > bp_filtered

# combine_results.py [-h] [--outf OUTF] VAR BP
python /home4/$USER/software/SortaDate/src/combine_results.py var_length_filtered bp_filtered --outf combined_results

#2d. Sort and get the list of the good genes with get_good_genes.py
# get_good_genes.py [-h] [--order ORDER] [--max MAX] [--outf OUTF] COMB
# --order gives the sorting order, where 1=root-to-tip variance, 2=treelength, 3=bipartition 
# 25 best loci

# 100 best loci
python /home4/$USER/software/SortaDate/src/get_good_genes.py combined_results --max 100 --order 3,1,2 --outf SortaDate_genes100

echo "All specified alignment files have been copied to $ALIGNMENTS_OUTPUT_DIR1"

#SBATCH --time=1:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --mem=1GB
#SBATCH --job-name=SortaDate

# Run this from 6_Phylogeny_ML/Paragone-PF

# Directorues and paths
# Update paths to match datasets 
ALIGNMENTS_DIR="/scratch/$USER/Hawaii_Mascarenes_Asteraceae/data/4_Alignments/Paragone/PF"
SORTADATE_DIR="/scratch/$USER/Hawaii_Mascarenes_Asteraceae/data/6_Phylogeny_ML/Paragone-PF/SortaDate"
ALIGNMENTS_OUTPUT_DIR1="$SORTADATE_DIR/Alignments25"
ALIGNMENTS_OUTPUT_DIR2="$SORTADATE_DIR/Alignments100"

# Create output directory for the alignments (for Step 3)
mkdir -p "$ALIGNMENTS_OUTPUT_DIR1"
mkdir -p "$ALIGNMENTS_OUTPUT_DIR2"

# Step 1. Root gene trees

# load phyx 
module load phyx/1.3-foss-2022a

cd Rooted_gene_trees/

# Root gene trees 
# Be sure to update to correct path to match dataset! 
# Update the outgroup as needed - Here we root between Asteraceae and Calyceraceae
for file in ../../../5_Phylogeny_coalescent/Paragone/Gene_trees/PF/RAxML_bipartitions.*_paragonePF;
do
    filename=$(basename "$file")  # Extracting just the file name without the path
    pxrr -t "$file" -g Calyceraceae_Calycera_crassifolia_ERR7618534,Calyceraceae_Boopis_anthemoides_ERR7618533,Calyceraceae_Acicarpha_tribuloides_ERR4180184 -o "rooted_${filename}" -s
done

# Step 2. Sorta date

cd ../SortaDate/
module load Python/3.9.6-GCCcore-11.2.0

#2a. Get the tip-to-root variation with the get_var_length.py
# get_var_length.py DIR [-h] [--flend FILEENDING] [--outg OUTGROUPS] [--loc PHYX_LOCATION] [--outf OUTF]

python /home4/$USER/software/SortaDate/src/get_var_length.py ../Rooted_gene_trees/ --flend _paragonePF --outg Calyceraceae_Calycera_crassifolia_ERR7618534,Calyceraceae_Boopis_anthemoides_ERR7618533,Calyceraceae_Acicarpha_tribuloides_ERR4180184 --outf var_length

#2b. Get the bipartition support with get_bp_genetrees.py
# get_bp_genetrees.py [-h] [--flend FILEENDING] [--loc PHYX_LOCATION] [--outf OUTF] DIR SPECIESTREE
# note here: species tree is wASTRAL with min at 0.33
python /home4/$USER/software/SortaDate/src/get_bp_genetrees.py ../Rooted_gene_trees/ --flend _paragonePF ../../../5_Phylogeny_coalescent/Paragone/Species_tree/PF/Species_tree_paragonePF_wastral.tre --outf bp

# Deal with problematic loci
# Several loci pose problems for combing in the results: in var_length some loci are NA; in bp some loci are 0
# We will filter out these loci in order for combine_results.py to run smoothly
awk '/NA/ {print $1}' var_length > skipped_genes.txt
awk 'NR==FNR {skipped[$1]; next} !($1 in skipped)' skipped_genes.txt var_length > var_length_filtered
awk 'NR==FNR {skipped[$1]; next} !($1 in skipped)' skipped_genes.txt bp > bp_filtered

#2c. Combine the results from these two runs with combine_results.py
# combine_results.py [-h] [--outf OUTF] VAR BP
python /home4/$USER/software/SortaDate/src/combine_results.py var_length_filtered bp_filtered --outf combined_results

#2d. Sort and get the list of the good genes with get_good_genes.py
# get_good_genes.py [-h] [--order ORDER] [--max MAX] [--outf OUTF] COMB
# --order gives the sorting order, where 1=root-to-tip variance, 2=treelength, 3=bipartition 
# 25 best loci
python /home4/$USER/software/SortaDate/src/get_good_genes.py combined_results --max 25 --order 3,1,2 --outf SortaDate_genes25
# 100 best loci
python /home4/$USER/software/SortaDate/src/get_good_genes.py combined_results --max 100 --order 3,1,2 --outf SortaDate_genes100

# Step 3. Extract loci alignments for the SortaDate loci
# Copy alignment files of the best genes

# 25
while IFS= read -r line; do
    # Skip the header line
    [[ "$line" == name* ]] && continue

    # Extract the gene name from the first column
    gene_name=$(echo "$line" | awk '{print $1}')

    # Remove the prefix "rooted_RAxML_bipartitions." from the gene name
    gene_id=${gene_name#rooted_RAxML_bipartitions.}

    # Remove the suffix "_paragonePF" from the gene name
    gene_id=${gene_id%_paragonePF}

    # Construct the alignment file path with ".selected_stripped.aln.trimmed.fasta" added to match alignment files
    alignment_file="$ALIGNMENTS_DIR/${gene_id}.selected_stripped.aln.trimmed.fasta"

    # Check if the alignment file exists and copy it to the output directory, or print a warning if it doesn't exist
    [[ -f "$alignment_file" ]] && cp "$alignment_file" "$ALIGNMENTS_OUTPUT_DIR1/" || echo "Warning: Alignment file for $gene_id not found."

done < SortaDate_genes25


# 100
while IFS= read -r line; do
    # Skip the header line
    [[ "$line" == name* ]] && continue

    # Extract the gene name from the first column
    gene_name=$(echo "$line" | awk '{print $1}')

    # Remove the prefix "rooted_RAxML_bipartitions." from the gene name
    gene_id=${gene_name#rooted_RAxML_bipartitions.}

    # Remove the suffix "_paragonePF" from the gene name
    gene_id=${gene_id%_paragonePF}

    # Construct the alignment file path with ".selected_stripped.aln.trimmed.fasta" added to match alignment files
    alignment_file="$ALIGNMENTS_DIR/${gene_id}.selected_stripped.aln.trimmed.fasta"

    # Check if the alignment file exists and copy it to the output directory, or print a warning if it doesn't exist
    [[ -f "$alignment_file" ]] && cp "$alignment_file" "$ALIGNMENTS_OUTPUT_DIR2/" || echo "Warning: Alignment file for $gene_id not found."

done < SortaDate_genes100

6a. Phylogeny - ML concatenation: Gene Shopping (Paragone-MO)

 “Gene shopping” is where phylogenomic dataset is winnowed to a set of genes with desirable properties.

 Here we run SortaDate (Smith 2018) to select the 25 and 100 most informative loci base on the following criteria
 1. Gene trees were ﬁltered by their similarity to the species tree
 2. clock-likeness (as indicated by root-to-tip variance)
 3. tree length.

 Paper: https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0197433
 Github: 

Approach implemented in Joyce 2023 (https://doi.org/10.3389/fpls.2023.1063174) and Zuntini 2024 (https://doi.org/10.1038/s41586-024-07324-0) for example. 

SortaDate requires phyx, which is already installed. Run the following to install SortaDate into python virtual environment:
# Load Python Module
module load Python/3.9.6-GCCcore-11.2.0
# Activate Virtual Environment
source /home4/$USER/software/venvs/target_capture/bin/activate
# Clone SortaDate Repository: Download the SortaDate package from GitHub
git clone https://github.com/FePhyFoFum/SortaDate.git
cd SortaDate
# Check installation
python src/get_var_length.py -h

SortaDate has two input requirements:
* a species tree topology must be provided 
* individual gene trees must be rooted (this is typically performed using outgroups)

This script is doing the following:
1. Root gene trees 
2. Run SortaDate to winnow down to the 25/100 most informative loci
3. Extract alignments for the 25/100 loci

#SBATCH --time=3-00:00:00

#SBATCH --cpus-per-task=2

#SBATCH --output=phylogeny-ML_hybpiper.out
#SBATCH --error=phylogeny-ML_hybpiper.er

# Run 6_Phylogeny_ML/Hybpiper/Phylogeny_ML

# Step 1. Concatenate the alignmenst for loci selected with SortaDate

# Load Python and activate the virtual environment with AMAS installed 
module load Python/3.9.6-GCCcore-11.2.0
source /home4/$USER/software/venvs/target_capture/bin/activate

# Run AMAS to concatenate the alignments
# 25
python3 /home4/$USER/software/venvs/target_capture/lib/python3.9/site-packages/amas/AMAS.py concat \
    --in-files ../SortaDate/Alignments25/*_aligned_rm.fasta \
    --in-format fasta \
    --data-type dna \
    --part-format raxml \
    --concat-out concatenation_hybpiper25.fasta \
    --concat-part partitions_hybpiper25.txt \
    --out-format fasta
# 100
python3 /home4/$USER/software/venvs/target_capture/lib/python3.9/site-packages/amas/AMAS.py concat \
    --in-files ../SortaDate/Alignments100/*_aligned_rm.fasta \
    --in-format fasta \
    --data-type dna \
    --part-format raxml \
    --concat-out concatenation_hybpiper100.fasta \
    --concat-part partitions_hybpiper100.txt \
    --out-format fasta

# Deactivate the virtual environment
deactivate

# Step 2. Build ML phylogeny with concatenated matrix

# Load RAxML with AVX2 support
module load RAxML/8.2.12-gompi-2021b-hybrid-avx2

# Run RAxML with optimizations and checkpointing
# 25
raxmlHPC-HYBRID-AVX2 -f a \
    -s concatenation_hybpiper25.fasta \
    -n phylogeny_ML_hybpiper25 \
    -m GTRGAMMA \
    -T 16 \
    -N 1000 \
    -k \
    -x 12345 \
    -p 67890 \
    -q partitions_hybpiper25.txt \
    -C checkpoint_file25
# 100
raxmlHPC-HYBRID-AVX2 -f a \
    -s concatenation_hybpiper100.fasta \
    -n phylogeny_ML_hybpiper100 \
    -m GTRGAMMA \
    -T 16 \
    -N 1000 \
    -k \
    -x 12345 \
    -p 67890 \
    -q partitions_hybpiper100.txt \
    -C checkpoint_file100

# Notes on the parameters
# -s [sequence file]
# -n [output file name]
# -m [substitution model][GTRCAT for faster analysis]
# -f a [bootstrap search and ML search in one analysis]
# -N autoMRE [automatically determine the number of bootstraps needed]
# -x [random seed for bootstrap]
# -p [random seed for parsimony inferences]
# -T [number of threads, set to available cores]
# -k [enables rapid bootstrap analysis]
# -q [file with the partitions]
# -C [checkpoint file name] to enable checkpointing

7. Dated Phylogeny 

Date ML trees with RelTime. 

https://bmcgenomics.biomedcentral.com/articles/10.1186/s12864-022-09030-5
https://academic.oup.com/gbe/article/10/6/1631/5034435
https://academic.oup.com/mbe/article/35/7/1770/4947869

For the Asteraceae-wide phylogeny we calibrated the tree with 10 fossils reviewed in Palazzesi et al. (2022) [https://doi.org/10.1093/botlinnean/boac032]
- Fossil calibration database with notes and justification at /data/7_Dated_phylogeny

RelTime is implemented in MEGA for Hybpiper, MO, PF maximum likelihood trees based on the a concatenated dataset of the 100 most informative loci

Each dataset directory has the following:
* Input ML tree 
* Calibration file
* Dated phylogeny 
* MEGA session file
