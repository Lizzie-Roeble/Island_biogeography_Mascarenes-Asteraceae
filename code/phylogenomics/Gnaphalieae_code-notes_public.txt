Step 1: Trim and clean the reads

For each of the 3 datesets (Gnaphalieae_islands, Gnaphalieae_Carme1, Gnaphalieae_Carme2), run trim through Hybpiper assembly seperately. 

A few things are happening in this script:
* making a new 1_trimmed directory (if needed)
* copying the raw files to the trimmed folder to do the work there
* change the file ending, I want to end up with *R1/2.fastq.gz so I need to remove everything until that
* unzip the files
* run trimmomatic on each sample
* organize the resulting samples: move sequences into paired and unpaired directories, and make a new dir for the unzipped sequences for reference

Code tips: 
* to check the unique file endings for the raw sequences, use: find . -type f -name '*_001_*.fastq.gz' | awk -F"_001_" '{print $2}' | sort -u

Notice: the unzipped trimmed reads take up a lot of storage. So after reads are assembled/mapped with Hybpiper delete from 1_trimmed from cluster and dont back up (easy to rerun if needed)

#!/bin/bash

#SBATCH --time=2-00:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --mem=6GB
#SBATCH --job-name=trimmomatic_batch_islands

# Start: Run this from the 0_raw_sequences/Gnaphalieae_islands directory

# Copy all the files into the trimmed reads directory
mkdir -p ../../1_trimmed/
cp *.filt.fastq.gz ../../1_trimmed/

# move into the trimmed reads directory
cd ../../1_trimmed/

# remove the filter extension (.filt.)
rename _001_AHGTCYDMXY.filt.fastq.gz .fastq.gz *_001_AHGTCYDMXY.filt.fastq.gz
rename _001_AHNC5FDSX7.filt.fastq.gz .fastq.gz *_001_AHNC5FDSX7.filt.fastq.gz
rename _001_AHNNH2DMXY.filt.fastq.gz .fastq.gz *_001_AHNNH2DMXY.filt.fastq.gz
rename _001_AHWGF5DSX7.filt.fastq.gz .fastq.gz *_001_AHWGF5DSX7.filt.fastq.gz
rename _001_BHT35KDRX2.filt.fastq.gz .fastq.gz *_001_BHT35KDRX2.filt.fastq.gz
rename _001_BHW5NHDSX7.filt.fastq.gz .fastq.gz *_001_BHW5NHDSX7.filt.fastq.gz
rename _001_BHWGWCDSX7.filt.fastq.gz .fastq.gz *_001_BHWGWCDSX7.filt.fastq.gz
# This is updated to the run specific file name, but generalized command:
# rename .filt.fastq.gz .fastq.gz *.filt.fastq.gz

# unzip the zipped sequence files
for zip in *.fastq.gz; do
    gunzip "$zip"
done

# Create paired and unpaired directories
mkdir -p paired unpaired

# load trimmomatic
module load Trimmomatic

# run trimmomatic
Total=$(ls *_R1.fastq | wc -l)

for i in $(eval echo {1..$Total}); do
    seq1=$(ls *_R1.fastq | head -n $i | tail -1)
    seq2=$(echo $seq1 | sed 's/_R1/_R2/g')
    name=$(basename $seq1 _R1.fastq)
    
    java -jar $EBROOTTRIMMOMATIC/trimmomatic-0.39.jar PE -threads 4 -trimlog Trimlog.txt $seq1 $seq2 \
        ${name}_1P.fastq ${name}_1U.fastq \
        ${name}_2P.fastq ${name}_2U.fastq \
        ILLUMINACLIP:$EBROOTTRIMMOMATIC/adapters/TruSeq3-PE-2.fa:2:30:10:2:true LEADING:10 TRAILING:10 SLIDINGWINDOW:4:20 MINLEN:40

    # Move the trimmed files to the appropriate folders
    mv ${name}_1P.fastq paired/
    mv ${name}_2P.fastq paired/
    mv ${name}_1U.fastq unpaired/
    mv ${name}_2U.fastq unpaired/

    # Delete unnecessary files to free up storage space
    rm ${name}_R*.fastq
done

Step 2b: Get summary stats from Hybpiper 

Get summary stats pre-filtering.

mkdir data/2_Hybpiper_summary

Breakdown of the script:
* datadir=$PWD stores the location to the Hybpiper data (so run this script from the 2_Hybpiper directory)
* Change current working directory to local disk ($TMPDIR) in compute node for faster processing  
* For loop to extract Hybpiper tar files to $TMPDIR. 
   * First line in loop interates of all the .tar files in the "datadir" directory (ie 2_Hybpiper)
   * "tar xf $datadir/$samplename.tar" extracts the contents of the .tar file specified by $samplename to the current working directory ($TMPDIR)
* Then we get the summary stats. 
   * hybpiper stats command will then summarize target enrichment and gene recovery efficiency for a set of samples
      * We need to supply the reference file used for mapping the reads. The file is located in the target_files directory, which is one level up from the current datadir directory.
      * And specify the path to the namelist.txt file that contains the names of the samples. The file is located in the directory one level up from datadir.
   * hybpiper recovery_heatmap takes the sequence length table output of hybpiper stats and creates a figure to visualize the recovery efficiency. 
   * Then copy the summary results back to /scratch in a new directory called 2_Hybpiper_summary

Resources and references:
* https://github.com/mossmatters/HybPiper/wiki#hybpiper-stats

form /scratch/$USER/Gnaphalieae/data/2_Hybpiper run:
sbatch /scratch/$USER/Gnaphalieae/scripts/hybpiper_summary_stats.sh
--------------------------------------------------------------------------------
#!/bin/bash

#SBATCH --time=2-00:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --mem=20GB
#SBATCH --job-name=hybpiper_summary_stats
ulimit -n 16384

# Run this script from the 2_Hybpiper folder

# Store location of data
datadir=$PWD

# load HybPiper2 module
module load MAFFT
module load Hybpiper/2.1.6-foss-2022b

# Change to local disk in compute node for faster processing
cd $TMPDIR

# Extract tar files to $TMPDIR
for samplename in $datadir/*.tar; do
  tar xf $samplename
done

# Summary stats

# hybpiper stats command will then summarize target enrichment and gene recovery efficiency for a set of samples. We just need to supply the target file and namelist.
hybpiper stats -t_dna $datadir/../target_files/comp1061_hybpiper_reference_fixed.fasta gene $datadir/../namelists/namelist_islands_carme1.txt --seq_lengths_filename seq_lengths_gnaphalieae1 --stats_filename hybpiper_stats_gnaphalieae1

# This command takes the sequence length table output of hybpiper stats and creates a figure to visualize the recovery efficiency.
hybpiper recovery_heatmap seq_lengths_gnaphalieae1.tsv --figure_length 40 --figure_height 15 --heatmap_filename recovery_heatmap_gnaphalieae1 --heatmap_filetype pdf
# Note! Because there are so many loci with the comp1061, we need to set the figure dimensions for the plot to be produced.

# copy back the results 
cp *.pdf *.tsv $datadir/../2_Hybpiper_summary

#!/bin/bash

#SBATCH --time=2-00:00:00
#SBATCH --nodes=1
#SBATCH --ntasks=1
#SBATCH --mem=20GB
#SBATCH --job-name=hybpiper_retrieve_sequences_paralogs

# Change to local disk in compute node for faster processing
cd $TMPDIR

# Extract tar files to $TMPDIR
for samplename in $datadir/*.tar; do
  tar xf $samplename
done

# Retrieve sequences 

# make the directory for the sequence outputs
# -p flag creates a directory if doesnt exist, ignores if does
mkdir -p $datadir/../2_Hybpiper_sequences

# This command will get sequences recovered for the same gene from multiple samples and generates an unaligned multi-FASTA file for each gene.
hybpiper retrieve_sequences dna -t_dna $datadir/../target_files/comp1061_hybpiper_reference_fixed.fasta --sample_names $datadir/../namelists/namelist_islands_carme.txt --fasta_dir $datadir/../2_Hybpiper_sequences
# Note! Use the namelist_islandPAFTOL_filtered.txt based on review of summary stats

# Recover paralogs 

# Recover the coding sequences from alternative long paralogs 
hybpiper paralog_retriever $datadir/../namelists/namelist_islands_carme.txt -t_dna $datadir/../target_files/comp1061_hybpiper_reference_fixed.fasta

# Outputs include:
# (1) a report that tabulates the number of sequences for each gene and sample (i.e. potential paralogs if > 1) 
# (2) heatmap visualisation
# (3) paralogs above threshold report
# (4) folder containing paralogs (5) directory containing paralogs with putative chimeras removed

# Copy all the paralog outputs back to the project directory in /scratch
mkdir -p $datadir/../2_Hybpiper_paralogs 

cp paralog_report.tsv $datadir/../2_Hybpiper_paralogs
cp paralog_heatmap.png $datadir/../2_Hybpiper_paralogs
cp paralogs_above_threshold_report.txt $datadir/../2_Hybpiper_paralogs
cp -r paralogs_no_chimeras/ $datadir/../2_Hybpiper_paralogs
cp -r paralogs_all/ $datadir/../2_Hybpiper_paralogs

Step 3. Paralogy resolution (Paragone) 

ParaGone is a Python package (with external dependencies) designed to identify ortholog groups from a set of paralog sequences from multiple taxa. This process has been decribed as both paralogy resolution and orthology inference. ParaGone implements multiple different algorithms to infer ortholog groups. These algorithms are applied to gene trees that contain paralogs, and ortholog groups are inferred based on algorithm-specific parsing of the tree topologies. The pipeline uses algorithms described and implemented by Yang and Smith 2014. These approaches have since been adapted for target capture datasets. (from github)

Resources and references:
* https://github.com/chrisjackson-pellicle/ParaGone
* https://github.com/chrisjackson-pellicle/ParaGone/wiki/Tutorial
* https://github.com/chrisjackson-pellicle/ParaGone/wiki
* https://academic.oup.com/mbe/article/31/11/3081/2925722

Step 5b. Build coalesent species tree with ASTRAL (Hybpiper)

The main steps:
1. Concatenate the gene tree files 
2. Run ASTRAL 

Build species tree with Weighted ASTRAL (wASTRAL) - Hybrid (default) 
* Threshold-free weighting schemes for the quartet-based species tree inference, the metric used in the popular method ASTRAL. 
*  By reducing the impact of quartets with low support or long terminal branches (or both), weighting provides stronger theoretical guarantees and better empirical performance than the unweighted ASTRAL
* https://github.com/chaoszhang/ASTER/blob/master/tutorial/wastral.md

wASTRAL outputs:
* the species tree topology
* branch lengths in coalescent units for astral-weighted and in combined (coalensecent + 2 * substitution) units (only for internal branches)
* branch supports measured as local posterior probabilities

The main steps:

2. Filter out the paralog ids so that the paralogs of the same sample are forced monophyletic

Build species tree with ASTRALpro
   * Can handle multiple copy genes
   * https://github.com/chaoszhang/ASTER/blob/master/tutorial/astral-pro3.md
   * https://doi.org/10.1093/bioinformatics/btac620
   * https://doi.org/10.1093/molbev/msaa139

ASTRALpro input:
 * The input gene trees are in the Newick format
 * The input trees can have missing taxa, polytomies (unresolved branches), and multi-copy genes
 * When multiple genes from the same species are available, you ask ASTRAL to force them to be together in the species tree via a mapping file or identical species names (strip paralogs ids - .main, .0, .1)

ASTRALpro outputs:
 * the species tree topology
 * branch lengths in substitution-per-site units (IQ-TREE like) for all branches
 * branch supports measured as local posterior probabilities

Build gene trees for each loci alignment generated through Paragone 

The more loci and more samples included in the project greatly increase run time, making a simple loop not the most efficient or cost effective way to build the individual gene trees. 

So we set the job up as an array. 
There are a couple small but important considerations:
* RAxML will often add a .fasta.reduced file back to the alignment directory, so the safest strategy is to define the alignment paths and files for a given dataset to a txt file right from the start and use this in the array
* Habrok has a limit of 1000 active jobs per user, so we will run the loci in two seperate batches 

The work flow: 
1. Create text that lists the path and file of all the alignments in a given dataset 
2. Run array batch 1
3. Run array batch 2
4. Check how all the jobs in the array performed

Build species tree with Weighted ASTRAL (wASTRAL) - Hybrid (default) 
* Threshold-free weighting schemes for the quartet-based species tree inference, the metric used in the popular method ASTRAL. 
*  By reducing the impact of quartets with low support or long terminal branches (or both), weighting provides stronger theoretical guarantees and better empirical performance than the unweighted ASTRAL
* https://github.com/chaoszhang/ASTER/blob/master/tutorial/wastral.md

wASTRAL outputs:
* the species tree topology
* branch lengths in coalescent units for astral-weighted and in combined (coalensecent + 2 * substitution) units (only for internal branches)
* branch supports measured as local posterior probabilities

Tree Comparison: Robinson Foulds Distances

Robinson-Foulds (RF) distance is a metric used to compare the differences between two phylogenetic trees. It's a commonly used measure in computational biology and bioinformatics to assess the similarity or dissimilarity between trees. 

The RF distance measures the dissimilarity between two phylogenetic trees by counting the number of different splits (bipartitions) between the trees. A split in a tree divides the set of taxa (species or genes) into two non-overlapping groups. The RF distance essentially counts the number of splits that are present in one tree but not in the other.

The R script below computed the RF distances between the trees and outputs a heatmap plot to compare the trees, and we also perfomr PCA on the distance matrix to provide a visual representation of the overall similarity among the trees.

#!/usr/bin/env Rscript

# Adjusted Robinson Foulds Distance

# Load necessary libraries
library(phangorn)
library(ape)
library(phytools)
library(ggplot2)
library(vegan)
library(reshape2)
library(viridis)
library(ggrepel)

# Read the trees
# Be sure path is correct
tr1 <- read.tree("../Hybpiper/Species_tree/Species_tree_hybpiper_wastral.tre")
tr2 <- read.tree("../Paragone/Species_tree/PF/Species_tree_paragonePF_wastral.tre")
tr3 <- read.tree("../Paragone/Species_tree/MO/Species_tree_paragoneMO_wastral.tre")
tr4 <- read.tree("../Paralogs/Species_tree/Species_tree_paralogs_astralpro.tre")

# 1. Preparation 

## Root the trees
outgroup <- c("Athrixia_phylicoides_R14373", "Phagnalon_sordidum_GC2743", "Leysera_gnaphalodes_R14525", "Oedera_pungens_R14489")

tr1 <- root(tr1, outgroup = outgroup, resolve.root = TRUE)
tr2 <- root(tr2, outgroup = outgroup, resolve.root = TRUE)
tr3 <- root(tr3, outgroup = outgroup, resolve.root = TRUE)
tr4 <- root(tr4, outgroup = outgroup, resolve.root = TRUE)

# Check if the trees are rooted
cat("Is tr1 rooted?", is.rooted(tr1), "\n")
cat("Is tr2 rooted?", is.rooted(tr2), "\n")
cat("Is tr3 rooted?", is.rooted(tr3), "\n")
cat("Is tr4 rooted?", is.rooted(tr4), "\n")

# 2. Make a table of RFadj values

# Create an empty matrix with 4 rows and 4 columns
matrixinp = matrix(data=NA, nrow=4, ncol=4) 

# Display the empty matrix 
print(matrixinp)

# Make list of trees
tree_list <- list(tr1, tr2, tr3, tr4)

for(j in 1:length(tree_list)){ 
  for (i in 1:length(tree_list)) {
    tree1 <- tree_list[[i]]  
    tree2 <- tree_list[[j]]
    matrixinp[i,j] = RF.dist(tree1, tree2, check.labels = TRUE, normalize = TRUE, rooted = TRUE)
  }
} 

# Give the matrix column and row names
colnames(matrixinp) <- c("HybPiper", "ParaGone-PF", "ParaGone-MO", "Paralogs")
rownames(matrixinp) <- c("HybPiper", "ParaGone-PF", "ParaGone-MO", "Paralogs")

# View the matrix
matrixinp

# export the matrix as a csv
write.csv(matrixinp, "RFDistance_Gnaphalieae_coalescent.csv")

# Plot the heatmap

# Melt the matrix to long format
matrix_long <- melt(matrixinp)

# Filter out the lower triangle and the diagonal by setting lower triangle values to NA
matrix_long <- matrix_long[as.numeric(matrix_long$Var1) <= as.numeric(matrix_long$Var2), ]

# Plot the heatmap with viridis color scale
heatmap_plot <- ggplot(data = matrix_long, aes(x = Var1, y = Var2, fill = value)) +
  geom_tile() +
  scale_fill_viridis(name = "RF Distance", option = "D", direction = 1) +
  theme_minimal() + 
  theme(axis.text.x = element_text(angle = 45, vjust = 1, 
                                   size = 10, hjust = 1),
        axis.text.y = element_text(size = 10),
        title = element_text(size = 10)) +
  labs(x = 'Tree Comparison', y = 'Tree Comparison', title = 'Heatmap of Adjusted Robinson-Foulds Distances')

# Print the heatmap
print(heatmap_plot)

ggsave(heatmap_plot,
       filename = "heatmap_RFDistance_Gnaphalieae_coalescent.pdf", 
       device = "pdf", width = 5, height = 4, units = "in")


# 3. Visualize Robinson-Foulds values as a PCA
# Classical multidimensional scaling (MDS) of a data matrix. Also known as principal coordinates analysis:

pca <- pcoa(matrixinp, correction = "none", rn = NULL)

# Convert PCoA results to a data frame for plotting
pca_df <- as.data.frame(pca$vectors)
pca_df$Sample <- rownames(pca_df)  # Add sample names for labeling

# Plot PCA
pca_plot <- ggplot(pca_df, aes(x = Axis.1, y = Axis.2)) +
  geom_point(size = 3, aes(color = Sample)) +
  geom_text_repel(aes(label = Sample), size = 4, box.padding = 0.5, point.padding = 0.5, max.overlaps = Inf) +
  scale_color_manual(values = RColorBrewer::brewer.pal(n = length(unique(pca_df$Sample)), name = "Set1")) +  # Use a default ggplot2 color palette
  theme_bw() +  # Use theme_bw for a clean background
  theme(plot.margin = margin(10, 10, 10, 10),  # Ensure there's space around the plot
        axis.title = element_text(size = 14),
        axis.text = element_text(size = 12),
        legend.position = "none") +  # Remove legend
  labs(x = paste("PCoA Axis 1 (", round(pca$values$Relative_eigenvalues[1] * 100, 1), "%)", sep = ""),
       y = paste("PCoA Axis 2 (", round(pca$values$Relative_eigenvalues[2] * 100, 1), "%)", sep = ""),
       title = "PCA of Robinson-Foulds Distances")
pca_plot

# Save the PCA plot
ggsave(pca_plot, filename = "pca_RFDistance_Gnaphalieae_coalescent.pdf", device = "pdf", width = 6, height = 5, units = "in")
